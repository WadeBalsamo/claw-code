# Ollamacode Server - API Specification
# =========================================
# This document describes how a client discovers and connects to an
# Ollamacode server running on another device on the local network.
#
# Server: setup_ollamacode_server.sh (installs ~/bin/ollamacode)

## Server Setup
Run on the server device:
  curl -fsSL https://ollama.com/install.sh | sh
  bash setup_ollamacode_server.sh

The server installer:
- Configures Ollama with OLLAMA_HOST=0.0.0.0:11434 (LAN-accessible)
- Optionally installs systemd override for persistent settings
- Installs ~/bin/ollamacode launcher that:
  - Starts Ollama in session mode
  - Offers model selection (The Dirty Dozen + custom pull)
  - Launches claw-code with local Ollama models

## Client Discovery & Connection

### 1. Discovery via mDNS / Avahi (recommended)
The server advertises on _ollama._tcp / _ollamacode._tcp:
  avahi-publish -s "ollamacode-server" _ollamacode._tcp 11434

The client scans:
  avahi-browse _ollamacode._tcp -r 2>/dev/null | grep "hostname"

### 2. Manual Configuration
Client sets:
  OLLAMA_REMOTE_HOST=http://<server-ip>:11434

Or via TUI:
  ollamacode-client --setup → option 1

### 3. API Endpoints Used by Client

All endpoints are standard Ollama HTTP API, served on port 11434:

  GET  /v1/models
       → OpenAI-compatible model listing
       Returns: {"object":"list","data":[{"id":"...","object":"model",...}]}

  POST /api/show
       → Model details (architecture, context length, parameter count)
       Body:   {"name":"<model-tag>"}
       Returns: {"model_info":{"<arch>.context_length": N, ...}}

  GET/POST /api/ps
       → Currently loaded/running models
       Returns: {"models":[{"name":"...","size":N,"size_vram":N,...}]}

  POST /api/generate
       → Pre-load a model into memory (warm-up)
       Body:   {"model":"<tag>","prompt":"ping","stream":false,"options":{"num_predict":1}}

  POST /api/pull
       → Pull/download a model from registry
       Body:   {"name":"<tag>","stream":true}
       Returns: NDJSON stream of download progress

### 4. Claw-Code Integration
The client launches claw-code with:
  export OPENAI_BASE_URL="http://<server>:11434/v1"
  export OPENAI_API_KEY="ollama"
  claw --model "openai/<model-tag>" --permission-mode workspace-write

The openai/ prefix is REQUIRED because claw's model syntax validator
only accepts provider/model format (openai/<model>).

### 5. Firewall Requirements
- Port 11434 (TCP) must be open between client and server
- Server must have OLLAMA_HOST=0.0.0.0:11434 (default from installer)
- No authentication is required (Ollama has no built-in auth)
